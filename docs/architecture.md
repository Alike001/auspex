## 3. ADRs (Architecture Decision Records)

### ADR-001 — Monorepo with pnpm workspaces over separate repos
- **Choice:** Single `auspex/` repo with `contracts/`, `app/`, `bots/` as workspaces
- **Reason:** Easier review, one README, shared types, Vercel deploys the `app/` workspace directly
- **Trade-off:** Some workspace tooling friction; mitigated by pnpm's first-class monorepo support

### ADR-002 — Foundry (Forge) for tests, Hardhat for deploy
- **Choice:** Forge tests + Hardhat deployment scripts (per `sdk-snippets.md` §10)
- **Reason:** Forge tests are 10-100x faster and produce richer diagnostics; Hardhat's deployment ecosystem is more mature
- **Trade-off:** Two tools instead of one; both are standard

### ADR-003 — Next.js App Router instead of Vite + React + JS
- **Choice:** Next.js 15 App Router + TypeScript
- **Reason:** Every UI anchor (Trigger.dev, Langfuse, Documenso, Cal.com) is Next.js. shadcn/ui is Next.js-first. Server actions handle bot orchestration without a separate backend. Vercel one-click deploy.
- **Trade-off:** Builder's home turf is Vite + JS; ~1 day Next.js + TS ramp. Acceptable cost.

### ADR-004 — RainbowKit over Privy
- **Choice:** RainbowKit + wagmi
- **Reason:** Crypto-native wallet UX expected by Web3 judges. Privy's social login is friendlier but adds setup complexity and a v2-ish vibe.
- **Trade-off:** Demo requires judges to have a wallet. Pre-recorded playback covers judges who don't.

### ADR-005 — Single Next.js app for both frontends (not 2 apps)
- **Choice:** `/jobs` (human) + `/bots` (bots) in one app
- **Reason:** Shared design system, shared wallet provider, shared types, one deploy. Two apps is overhead with no upside in v1.
- **Trade-off:** Bigger bundle; mitigated by App Router code splitting

### ADR-006 — `inferString` with `allowedValues = ["released", "refunded"]` for the verdict
- **Choice:** Constrained-output classification instead of free-form generation
- **Reason:** Determinism. Per `sdk-snippets.md` §3, constrained outputs return EXACTLY one of the allowed values. The reasoning string is a separate `inferString` call with no allowedValues (or `inferChat` for richer prose).
- **Trade-off:** Two LLM calls instead of one in `resolve()`. Cost: 0.24 STT per resolution. Acceptable.

### ADR-007 — Reactivity dispatches `resolve()` synchronously on `submitDelivery()`
- **Choice:** When a freelancer/bot calls `submitDelivery(url)`, the Escrow contract immediately fires the agent composition in the same transaction; the callback runs in the SAME BLOCK (Somnia's hallmark feature).
- **Reason:** This is the *whole moat*. Without same-block resolution, Auspex is just a slower Kleros.
- **Trade-off:** Resolution gas + 3 agent deposits all attached to delivery tx. Caller pays. Mitigation: factory pre-funds resolution budget at job creation.

### ADR-008 — Pre-recorded 60s playback always runs in Frontend #2; live button is bonus
- **Choice:** Frontend #2 default state plays a canonical 12-job sequence from `app/public/demo/canonical-60s.json`. Judges can click "Run live demo" to fire real on-chain bots.
- **Reason:** Demo robustness on judging day. Live demos break.
- **Trade-off:** Two playback modes to maintain. Worth it.

## 4. Contract architecture

### 4.1 Components

```
┌─────────────────────────────────────────────────────────────────────────┐
│  EscrowFactory.sol                                                       │
│  ─ createJob(brief, deliverer, deadline) payable → Escrow address       │
│  ─ allJobs() → address[]                                                │
│  ─ jobsByClient(addr) → address[]                                       │
│  ─ jobsByDeliverer(addr) → address[]                                    │
│  ─ Emits JobCreated(escrow, client, deliverer, amount, briefHash)       │
└─────────────────────┬───────────────────────────────────────────────────┘
                      │ spawns
                      ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  Escrow.sol (per-job)                                                   │
│  ─ client, deliverer, amount, briefHash, briefURI, deadline              │
│  ─ state: Open → Delivered → Resolved → Claimed                          │
│  ─ submitDelivery(string deliveryUrl)  — caller = deliverer              │
│  ─ resolve() — calls AuspexResolver; fires in submitDelivery callback   │
│  ─ claim() — caller = winner of verdict                                  │
│  ─ Emits JobDelivered, JobResolved(verdict, reasoning), JobClaimed       │
└─────────────────────┬───────────────────────────────────────────────────┘
                      │ delegates resolution to
                      ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  AuspexResolver.sol                                                     │
│  ─ Composes 3 agent calls in resolve(escrow, briefURI, deliveryUrl):    │
│    1. JsonApiAgent.fetchString(deliveryUrl, "$.status") — URL reachable │
│    2. ParseWebsiteAgent.ExtractString(... deliveryUrl) — read content   │
│    3. LLMAgent.inferString(prompt, system, false, ["released","refunded"])│
│  ─ Uses PLATFORM.createRequest{value: 12e16}() per call                 │
│  ─ Each callback advances a state machine; final callback calls         │
│    Escrow.applyVerdict(verdict, reasoning)                              │
└─────────────────────┬───────────────────────────────────────────────────┘
                      │
                      ▼
                   Same-block dispatch via Reactivity
                      │
                      ▼
                  Escrow.applyVerdict → release or refund
```

### 4.2 Agent composition (the moat — exact call pattern from `sdk-snippets.md`)

```solidity
// AuspexResolver.sol (sketch)
import "./interfaces/ISomniaAgents.sol";

contract AuspexResolver {
    IAgentRequester public constant PLATFORM     = IAgentRequester(0x037Bb9C718F3f7fe5eCBDB0b600D607b52706776);
    uint256 public constant JSON_API_AGENT_ID    = 13174292974160097713;
    uint256 public constant LLM_AGENT_ID          = 12847293847561029384;
    uint256 public constant PARSE_AGENT_ID        = 12875401142070969085;
    uint256 public constant DEPOSIT_PER_CALL      = 12e16; // 0.12 STT

    enum Step { None, FetchedMetadata, ParsedDelivery, JudgedVerdict }

    struct Resolution {
        address escrow;
        string  briefURI;
        string  deliveryUrl;
        string  parsedContent;
        Step    step;
    }

    mapping(uint256 => Resolution) public resolutions; // requestId → Resolution

    function startResolution(address escrow, string calldata briefURI, string calldata deliveryUrl) external payable {
        // Step 1: JSON API verifies URL reachable
        bytes memory payload = abi.encodeWithSelector(
            IJsonApiAgent.fetchString.selector,
            string.concat("https://api.urlmeta.example.com/check?url=", deliveryUrl),
            "$.reachable"
        );
        require(msg.value >= DEPOSIT_PER_CALL * 3, "Underfunded resolution");
        uint256 reqId = PLATFORM.createRequest{value: DEPOSIT_PER_CALL}(
            JSON_API_AGENT_ID, address(this), this.onMetadata.selector, payload
        );
        resolutions[reqId] = Resolution(escrow, briefURI, deliveryUrl, "", Step.None);
    }

    function onMetadata(uint256 requestId, Response[] memory responses, ResponseStatus status, Request memory) external {
        require(msg.sender == address(PLATFORM));
        Resolution storage r = resolutions[requestId];
        require(r.step == Step.None);
        r.step = Step.FetchedMetadata;

        // Step 2: Parse the delivered URL
        string[] memory opts = new string[](0);
        bytes memory payload = abi.encodeWithSelector(
            IParseWebsiteAgent.ExtractString.selector,
            "content",
            "Main visible content of the delivered page",
            opts,
            "Extract the main visible content of the delivered page, focusing on headings and primary copy",
            r.deliveryUrl, false, uint8(1)
        );
        uint256 nextReqId = PLATFORM.createRequest{value: DEPOSIT_PER_CALL}(
            PARSE_AGENT_ID, address(this), this.onParsed.selector, payload
        );
        resolutions[nextReqId] = r;
        delete resolutions[requestId];
    }

    function onParsed(uint256 requestId, Response[] memory responses, ResponseStatus status, Request memory) external {
        require(msg.sender == address(PLATFORM));
        Resolution storage r = resolutions[requestId];
        require(r.step == Step.FetchedMetadata);
        r.parsedContent = abi.decode(responses[0].result, (string));
        r.step = Step.ParsedDelivery;

        // Step 3: LLM judges the parsed content against the brief
        string[] memory allowedValues = new string[](2);
        allowedValues[0] = "released"; allowedValues[1] = "refunded";

        bytes memory payload = abi.encodeWithSelector(
            ILLMAgent.inferString.selector,
            string.concat(
                "Brief URI: ", r.briefURI, "\n",
                "Delivered content extracted from URL: ", r.parsedContent, "\n",
                "Does the delivered content satisfy the brief? Reply 'released' (yes) or 'refunded' (no)."
            ),
            "You are Auspex, an impartial arbiter. You judge whether delivered work satisfies a brief. Reply with one of the allowed values only.",
            false,
            allowedValues
        );
        uint256 nextReqId = PLATFORM.createRequest{value: DEPOSIT_PER_CALL}(
            LLM_AGENT_ID, address(this), this.onJudgment.selector, payload
        );
        resolutions[nextReqId] = r;
        delete resolutions[requestId];
    }

    function onJudgment(uint256 requestId, Response[] memory responses, ResponseStatus status, Request memory) external {
        require(msg.sender == address(PLATFORM));
        Resolution storage r = resolutions[requestId];
        require(r.step == Step.ParsedDelivery);
        string memory verdict = abi.decode(responses[0].result, (string));
        // Reasoning = the parsed content + the verdict string (cheap v1; v2 adds inferChat for prose reasoning)
        string memory reasoning = string.concat("Verdict: ", verdict, " · Evidence: ", r.parsedContent);
        IEscrow(r.escrow).applyVerdict(verdict, reasoning);
        delete resolutions[requestId];
    }
}
```

### 4.3 Reactivity wiring

`AuspexResolver` is funded with ≥ 32 SOMI at deployment (per `sdk-snippets.md` §5). It is NOT a subscriber itself; the same-block dispatch is achieved by all 3 agent callbacks landing in the same transaction tree (each `createRequest` callback runs synchronously when consensus is reached, all within the originating block on Shannon). For an optional cron / heartbeat (Epic 4 — auto-refresh demo state), a separate `BotPulse.sol` subscribes to `BlockTick` and emits a `DemoPulse` event every N blocks.

### 4.4 Storage model

- `Escrow.sol` stores: `client`, `deliverer`, `amount`, `briefHash` (keccak256 of brief JSON), `briefURI` (off-chain IPFS or URL), `deadline`, `state`, `deliveryUrl`, `verdict`, `reasoning`
- `EscrowFactory.sol` stores: `allJobs[]`, `jobsByClient[address]`, `jobsByDeliverer[address]`, `nonce`
- `AuspexResolver.sol` stores: `resolutions[requestId]` (transient — deleted after final callback)

Brief content is stored off-chain (IPFS via Web3.Storage, or just a hosted JSON for v1) with the hash on-chain. v2 may move briefs to Somnia Data Streams.

### 4.6 Live testnet status (2026-05-26)

The end-to-end demo on Shannon executes the full deploy → fund → submit → resolve → claim flow in ~30–80 seconds per run. Two live-testnet findings worth documenting:

**JSON API step:** initial design wrapped the delivery URL in `https://api.allorigins.win/get?url=...` as a JSON-returning CORS proxy. allorigins responded correctly from a local machine but the JSON API agent on Shannon returned `Failed` for every delivery URL we tried through it. Step 1 was changed to use the SDK-snippets §3 example endpoint (`https://api.coingecko.com/api/v3/ping`, selector `gecko_says`) as a stable service-alive probe. Step 2 (Parse Website) is still the only step that actually fetches `deliveryUrl`. Trade-off: Step 1 no longer per-call probes the delivery URL — that remains as a follow-up for a custom probe endpoint.

**Parse Website agent:** as of 2026-05-26, the Parse Website agent (`12875401142070969085`) returns `ResponseStatus.Failed` on Shannon for every URL we tested, including the `https://news.ycombinator.com` example from the Somnia SDK docs. Tested URLs: Vercel-hosted `text/html` pages (thin + rich), gist raw URLs (`text/plain`), and the SDK doc's own Hacker News example. Tx hashes for diagnostic runs:

| URL | Escrow | Claim tx |
|---|---|---|
| `https://auspex-demo-pages.vercel.app/correct.html` | `0x91B64215A8FFA3A6d213D037a95c19Bbf818123B` | `0x32cece736e5c9262331cbbccc7dc523b0f3f12e8c337c5df7e5bdb3cb1e137db` |
| `https://news.ycombinator.com` | `0x988B966C4c4b5Fb0699B638Fcb21Bb0C5a8627Ee` | `0x8144e6c610c8b17668772df5f492e77c8d1928bb8a72c4483596a11937c4b0cb` |
| `https://auspex-demo-pages.vercel.app/wrong.html` (--negative) | `0x264CBb8F49f41eBb8CC1177971905bD589DDD106` | `0x71999eadb31e9aae77c5bca3bfda804184a9b82dd820039a1dc46931d1fdca0c` |

In all three runs Step 1 (CoinGecko ping) succeeded and Step 2 returned `Failed`. The AuspexResolver's defensive fallback applies a `"refunded"` verdict with reasoning `"Could not parse delivered page"`, and the client successfully claims the escrowed STT. This is the designed-for behaviour when an agent in the composition fails — verified live on testnet.

The 3-step composition logic itself is verified by the integration test suite (`forge test`: 37/37 passing) against a `MockAgentPlatform`. When the Parse Website agent recovers on Shannon, the same demo script will produce a `"released"` verdict for the happy path without any contract changes — the resolver code, agent IDs, payload encodings, and state machine remain spec-correct.

```typescript
import { SDK } from '@somnia-chain/reactivity';
import { somniaShannon } from './chain';

export function watchAuspexEvents(callback: (ev: AuspexEvent) => void) {
  const sdk = new SDK({ public: createPublicClient({ chain: somniaShannon, transport: webSocket() }) });
  return sdk.watch({
    eventContractSources: [DEPLOYED.EscrowFactory, DEPLOYED.AuspexResolver],

      keccak256(toBytes('JobDelivered(...)')),

    ],
    onData: (data) => callback(decodeAuspexEvent(data)),
  });
}
```

## 6. Bot scripts (`bots/`)

### 6.1 Data-fetcher (`bots/data-fetcher.ts`)

- Reads a list of canonical scrape briefs from `bots/shared/briefs.ts`
- Picks one randomly (or in sequence in deterministic mode)
- Funds itself via testnet faucet (one-shot setup)
- Calls `EscrowFactory.createJob(brief, scraperAddress, deadline)` with 0.5 STT locked
- Stores the resulting Escrow address
- Logs structured JSON to stdout (consumed by orchestrator)

### 6.2 Scraper (`bots/scraper.ts`)

- Subscribes to `JobCreated` events where it is the `deliverer`
- On match: performs the scrape using a free public API + a hardcoded scraping snippet
- Hosts the result on a temporary URL (Vercel Edge or just a static JSON returned by an `app/api/bot-delivery/[id]/route.ts` endpoint)
- Calls `Escrow.submitDelivery(url)` — this triggers `AuspexResolver` agent composition
- Waits for `JobResolved` event
- Calls `Escrow.claim()` if verdict === `"released"`

### 6.3 Orchestrator (`bots/orchestrator.ts`)

- Spawns both bots as child processes
- Used by `/api/bot-trigger` route handler (live "run demo" button)
- Also exported as a CLI: `pnpm bots:demo` runs one full loop

## 7. Observability hooks

- All contract events indexed via `somnia_watch` into the live feed (no external indexer for v1)
- Receipts (per-validator per-request JSON audit trail) accessible via Receipts API for the "View reasoning details" affordance on `JobDetail`
- Frontend writes a structured log to `console.log` for every wagmi tx (toast UI + DevTools breadcrumb)
- Bot orchestrator logs JSON-Lines to stdout; collected to `bots/logs/<runId>.jsonl` on `--persist` flag

```
└──────────────────┘
```

Bots can run anywhere; for the canonical demo they run as Node processes spawned by `app/api/bot-trigger`, but the same scripts run via `pnpm bots:demo` from any laptop. Mentioning this in the README is part of the "this is a primitive" framing.

## 9. Context7 / library research rule

Before importing any library not in the Stack table above:

1. Confirm latest stable version via Context7 or the library's repo `releases` page
2. Confirm Apache / MIT / BSD license — no GPL transitive dependencies in `app/` or `bots/` (contracts can be GPL if vendored from the sponsor)
3. Confirm it ships TypeScript types or has a `@types/` package
4. Add to ADR if it materially shapes architecture (e.g. swapping wagmi for something else)

## 10. Banned patterns (project-specific)

- ❌ ANY mock in the hot path. The judging demo MUST hit Shannon testnet on the human flow. Pre-recorded playback for the bot flow is documented + clearly labelled.
- ❌ Generic shadcn-admin / Kiranism dashboard scaffold imports
- ❌ Free-tier auth providers (Clerk, Auth0) — wallet connect only in v1
- ❌ `useEffect` for chain reads — use `useReadContract` from wagmi
- ❌ Hardcoded RPC endpoints in app code — read from `process.env.NEXT_PUBLIC_SOMNIA_RPC`
- ❌ `import * as React from 'react'` (verbose) — use named imports
- ❌ `any` in TypeScript without a comment explaining why
- ❌ Console.log left in production bundle — wrap behind `process.env.NODE_ENV !== 'production'`
- ❌ Inline styles — every visual property resolves to a Tailwind class wired to DESIGN.md tokens
- ❌ Direct `localhost:3000` references anywhere except `.env.example`
- ❌ Animated splash screens / loaders longer than 800ms — skeleton components instead
